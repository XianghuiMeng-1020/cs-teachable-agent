# 与已公开对比文件的区别点及创造性初步分析

**文档用途**：供专利代理机构撰写意见陈述书、新颖性/创造性答复或申请策略参考。  
**对比文件**：CN119624716A —《基于大语言模型的虚拟教学训练方法、系统及介质》（申请公布号 CN119624716A，申请号 202411738589.5 等，以官方文本为准）  
**我方技术方案**：编程教育场景下的「可教智能体（Teachable Agent）」系统——人类学习者向大模型扮演的虚拟学习者施教，系统以**动态知识状态**驱动虚拟学习者行为，并与**代码结构校验、沙箱执行、任务调度、过程追溯**等形成闭环。

| 维度 | 对比文件公开内容（概括） |

## 二、我方方案（新补充内容）技术要点摘要

| 维度 | 我方方案要点（与交底书/增强版说明书一致） |
|------|------------------------------------------|
| **应用场景** | 面向**学习者（学生）**的编程教育；通过向虚拟学习者施教实现「以教促学」，侧重**知识内化与元认知** |
| **虚拟学习者起点** | **非预设能力档位**：以知识单元为粒度，从零或极低先验开始，随教学与测评**动态演化** |
| **知识建模** | **扩展知识状态**（含错误概念、证据链等），在传统 BKT 思路上引入与**错误概念形成/纠正**、**教学质量**、**前置依赖**相关的约束与更新 |
| **行为约束** | **知识状态约束的大模型输出**（如 Reflect–Respond：提取—反思存储—检索—严格边界提示生成），避免「全知」式回答 |
| **编程特有** | **AST（抽象语法树）级**将**语法节点类型**映射到**知识单元**，用于约束可生成/可执行代码与知识边界一致 |
| **执行闭环** | **沙箱执行** + 多类错误归因，结果**回写**知识状态与错误概念诊断 |
| **调度** | **多策略任务选择**（覆盖、难度、间隔、错误概念导向、不确定性等）与 **SM-2 类间隔重复与 BKT 掌握度联动**（若实际实现） |
| **可追溯** | **多类型事件**记录教学—状态—对话—选题—尝试—评估的因果关联，支持解释与诊断 |

## 三、区别点对照表（代理老师可直接引用）

下表按「区别主题 → 对比文件 → 我方 → 是否可能构成区别」整理。

| 序号 | 区别主题 | 对比文件已公开/倾向 | 我方补充/不同做法 | 与对比文件是否易区分 |
|------|----------|---------------------|-------------------|------------------------|
| D1 | **应用目的与用户角色** | 培训**教师**，专家系统给**教学策略建议** | **学生**向虚拟学习者施教，侧重**自身学习效果**；教师端可为**分析/管理**，核心闭环在「学—教—测—状态」 | **较易区分** |
| D2 | **虚拟学习者能力来源** | **预设**先验知识、认知、学习风格等参数生成虚拟学生 | **非预设档位**：以**知识单元 + 动态掌握概率/状态**演化能力边界；强调「随教学变化」 | **较易区分** |
| D3 | **多模态 vs 编程闭环** | 文本/图像/视频等多模态；偏通用学科材料 | **源代码结构 + 沙箱执行 + 测评**与知识状态**联动**；针对**编程**可验证性 | **较易区分**（领域与闭环不同） |
| D4 | **专家评估模块** | **独立虚拟专家**阶段性评估教学、给建议 | 我方核心是**状态机 + 约束生成 + 执行反馈**；**不依赖**「专家给教师建议」这一主路径（可有分析报表，但非同一发明点） | **较易区分** |
| D5 | **知识粒度与状态** | 「学习状态信息」较宏观；未公开**知识单元级 BKT+前置依赖+证据** | **细粒度知识单元**、**前置依赖门控**、**多类证据**（教学/测试/纠正等） | **中等**（需强调**结构化实现**与**联动**，避免被指为公知 BKT 常规应用） |
| D6 | **错误概念** | 未公开**错误概念激活、严重度、衰减、与测评联动** | **错误概念目录 + 失败尝试/规则/LLM 诊断 + 状态回写** | **中等偏强**（若权利要求写清**触发条件与状态联动**） |
| D7 | **遗忘与复习** | 未公开与本方案同构的**遗忘曲线 + 复习调度与掌握度融合** | **个性化衰减**（如与交互次数相关）+ **SM-2 与掌握度联动调整间隔** | **中等**（单独「遗忘曲线」易被指公知；**融合调度与门控**更易争创造性） |
| D8 | **AST 语法结构守卫** | 未涉及**源码 AST 与知识单元映射** | **节点类型 → 知识单元集合**的映射与**子集校验**，控制可执行代码与知识边界一致 | **较强**（编程教育场景下**具体技术手段**较具体） |
| D9 | **多策略任务选择** | 未公开**按错误概念/不确定性/间隔等策略切换** | **多策略打分与切换规则**（与掌握度、错误概念活跃度等相关） | **中等偏强**（需写清**输入特征与决策规则**） |
| D10 | **Reflect–Respond 四阶段** | 未公开「提取—反思存储—检索—**严格仅用已检索知识**的提示约束」这一**管道化**方案 | **分阶段**约束生成，明确**禁止越权知识**的提示结构 | **中等**（可能被质疑为**提示工程**；需绑定**状态数据源**与**可验证效果**） |
| D11 | **证据链 / 可追溯** | 未公开**十种级事件类型 + 前后状态快照 + 关联 ID** 的审计模型 | **教学—状态—对话—选题—尝试—评估**全链路事件与因果关联 | **中等**（日志系统易被指常规；**与知识更新强绑定**更关键） |
| D12 | **模式切换（苏格拉底提问）** | 未公开**按轮次切换提问者模式** | **周期性**切入 Why/How 类追问 | **偏弱至中等**（单独看易被指教学技巧常规实现） |

## 五、可能被审查员/客户指出的风险点（便于代理提前准备）

1. **「以教促学 + LLM 虚拟学生」**  
   - 风险：其他专利或论文已描述类似教学关系。  
   - 应对：将创新点锚定在 **D3、D8、D6、D7 的联动机制**与**可验证实现**，而非关系描述本身。

2. **「BKT、遗忘曲线、沙箱」各自为公知**  
   - 风险：容易被认定为**常规组合**。  
   - 应对：强调**组合后解决的具体技术问题**（例如：防止 LLM 超越当前知识边界输出、编程结果与知识状态一致、复习间隔随掌握度自适应），并尽量用**结构+步骤+参数关系**限定。

3. **「提示词/管道」创造性不足**  
   - 风险：D10 单独支撑弱。  
   - 应对：与 **状态矩阵、AST 允许集合、沙箱反馈**写进同一权利要求链条或方法步骤中。

4. **对比文件「学习状态信息」**  
   - 风险：审查员可能认为已涵盖广义状态更新。  
   - 应对：用 **D5/D6/D8** 的**数据结构、状态种类、更新触发源**细化差异，避免仅说「有学习状态」。

## 七、一句话结论（给客户的非法律结论）

- **与 CN119624716A 最容易拉开距离的区别点**：**编程场景下的 AST–知识单元绑定 + 沙箱执行结果驱动知识状态/错误概念**（D3、D8），以及 **动态知识边界约束生成与测评/选题闭环**（D2、D5、D9 的组合），而非单独强调「逆向教学」叙事。  
- **单独支撑创造性偏弱的点**：泛泛的 BKT、沙箱、遗忘曲线、日志、纯提示工程——需通过**联动设计与具体规则**补强。  
- **是否「有创造性」**：在**充分检索**前无法定论；从交底角度，**D3+D8+D6+D7 的协同**较有希望作为答复重点，**D1、D12**宜作辅助。
