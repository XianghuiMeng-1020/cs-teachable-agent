In this programming task, you'll work with the concept of teleportation commonly seen in science fiction. Imagine there is a futuristic city where a network of teleportation portals allows rapid transportation between various nodes. However, due to energy constraints, there are rules about which nodes can teleport to which based on availability.

You need to write a function called `can_teleport_sequence(nodes, connections)` that determines whether a series of teleportation requests can be fulfilled consecutively. Here's how the task is outlined:

- `nodes` is a list of integers representing the series of requested teleportations from one portal to another.
- `connections` is a list of integer pairs, where each pair (a, b) indicates a one-way teleportation portal that allows transportation from node `a` to node `b`.

The function should return `True` if all requested teleportations can be made consecutively from the start of the list to the end; otherwise, return `False`.

### Constraints
- The `nodes` list will have at least 2 elements.
- All numbers within the list represent valid node names.
- Node numbers and connections are strictly positive integers.

### Example
```python
can_teleport_sequence([1, 2, 3], [(1, 2), (2, 3)]) # Returns: True
can_teleport_sequence([1, 2, 3], [(1, 3), (3, 2)]) # Returns: False
```

Your goal is to implement this function such that when tested, it returns the correct boolean value for whether a complete series of teleportations is possible given the available connections.
